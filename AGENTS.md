## 前提

作業を開始する前に、タスクの内容を判断して必要なドキュメントを読み込むこと。

| タスク種別 | 読み込むドキュメント |
|-----------|------------------|
| プロダクト要件 / 仕様確認 / 機能スコープ判断 | `docs/app_overview.md` |
| ローカル環境構築 / Terraform 操作 / セットアップ関連 | `docs/setup_guide.md` |
| リポジトリ構成 / ディレクトリの役割把握 / 変更箇所の特定 | `docs/repository_layout.md` |
| 複数領域にまたがる場合 | 該当するすべてを読む |
| 上記に当てはまらない軽微な変更 | 読み込み不要 |

## 読み込み禁止対象 (厳守)

以下のファイルは、ユーザーから明示的に指示があった場合を除き、**読み込み・参照・引用してはならない**。タスクの判断・回答の根拠としても使用しないこと。

## 回答スタイル
- 結論を最初に述べること
- 根拠を明示すること
- 忖度せず、良い点・悪い点があれば正直に伝えること(なければ無理に取り繕って伝えなくてよい)
- 闇雲に回答せず、判断の文脈（トレードオフ・前提条件など）も合わせて伝えること
