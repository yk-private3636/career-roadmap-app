# career-roadmap-app 概要

## 1. プロダクト概要

自分のこれまでのキャリアを図式化(タイムラインコンポーネント等)して可視化し、共有できるアプリケーション。

共有された他者のキャリアは、「どんな経路を歩み、どこで転機が起きたか」を読み解くための参考情報としても活用できる。書く側(自分のキャリアを表現したい人)と読む側(他者のキャリア遷移を参考にしたい人)の双方に価値を置く。

## 2. ターゲットとユースケース

- **主用途: 面接時の自己紹介・キャリア説明ツール**
  - 面接時に使われる説明ツールのデファクトスタンダードを目指す。
- **共有用途: 他者のキャリアを参考情報として閲覧**
  - キャリアチェンジを検討中の人がロールモデルを探す。
  - 同職種・別職種の遷移パターンを読み解く。
- **派生用途: エンジニアの案件遍歴・スキルシート代替**
  - エンジニアであれば、案件遍歴をまとめたスキルシート的な使い方も可能。

## 3. 差別化ポイント

- 共通の図解フォーマットで、誰のキャリアも同じ見方で読める。
  - 個人ごとにフォーマットが違わないため、「他者と並べて比較・参考にする」という読み方に向く。
- LinkedIn 等の既存サービスは文章中心。本アプリは「図でひと目で伝わる」ことに重きを置く。

## 4. 主な機能 (初期スコープ)

- キャリア情報の登録・編集 (時系列のイベント / 期間 / 役割 / スキル等)
- タイムラインを中心とした図解での可視化
- 共有 (パブリック公開URL / プライベートグループ招待 — 詳細は §5)
- スライドショー表示 (登録キャリアを順送りでプレゼン表示)
  - 面接プレゼン用途と直結する一次機能として位置付ける。
- コンタクト手段の表示 (任意項目)

## 5. 公開範囲・プライバシー

アカウント単位で2種類の可視性を持つ。

### パブリック

- 公開URLで誰でも閲覧可能。
- 他者からの「キャリア参考用途」での閲覧を想定する基本モード。

### プライベート

- 既定では本人のみ閲覧。
- 「グループ」を作成し、招待制で一部のステークホルダーのみと共有できる。
- グループは目的別に複数作成可能 (例: 転職活動先A社用 / 社内上長用 / 友人用)。
- **グループごとに提示内容を出し分け**できる。
  - 同一プロフィールから、相手に応じて見せる範囲・粒度を変えられる。
- 都度ステークホルダーを区切って連携することを想定。

## 6. 認証

- OAuth のみ (自前で認証機能は持たない)。

## 7. 提供しない機能 (明示)

- アプリ内 DM / メッセージング機能。
  - コンタクトは、ユーザーが任意で記載した外部手段経由で行う。
